# 产研流程编排器（Dev Pipeline Orchestrator）

## 概述

标准产研流程有 7-8 个阶段，用户（尤其非技术背景）容易迷失——不知道当前在哪、下一步该做什么、需要谁参与。本 Skill 提供流程导航能力。

**核心原理**：
- 读取 MEMORY.md §2 获取当前流程位置
- 对照标准产研流程表，告诉用户：你在哪、下一步是什么、产出物是什么
- 每个阶段完成时自动提醒流转

---

## 标准产研流程

```
需求规划 → PRD撰写 → 设计评审 → 原型制作 → UI设计 → 技术方案 → 代码实现 → 测试验证 → 上线复盘
```

| 阶段 | 输入 | 产出物 | 关键角色 |
|------|------|--------|---------|
| 需求规划 | 用户痛点、场景 | 需求概要、优先级清单 | 产品 |
| PRD撰写 | 需求概要 | PRD 文档（EARS 原则） | 产品 |
| 设计评审 | PRD | 评审议题、交互逻辑清单 | 产品+设计+研发 |
| 原型制作 | 设计评审结论 | 可交互原型 | 设计 |
| UI设计 | 原型+设计规范 | 高保真 UI 稿 | 设计 |
| 技术方案 | UI 稿+PRD | 技术方案文档 | 研发 |
| 代码实现 | 技术方案+UI 稿 | 产品代码 | 研发 |
| 测试验证 | 代码+PRD | 测试用例、验收报告 | 测试+产品 |
| 上线复盘 | 测试报告 | 复盘报告、vNext 清单 | 全员 |

## 三条铁律

### 铁律 1：每阶段开始前确认输入

进入新阶段前，必须确认上一阶段的交付物已经就绪。

```
"当前在 [阶段A] → [阶段B]。
 [阶段B] 需要输入：[输入物]。
 请确认 [输入物] 已经完成，我们进入 [阶段B]？"
```

### 铁律 2：每阶段结束时提醒流转

阶段完成后，主动提醒：

```
"[阶段B] 已完成。
 交付物：[产出物清单]
 下一步：[阶段C]，需要 [角色] 参与。
 是否流转？"
```

### 铁律 3：不跳环

严格按流程走，不允许跳过任何阶段。如果用户想跳过，提醒风险：

```
"跳过 [阶段B] 直接进入 [阶段C] 的风险：
 1. [具体风险1]
 2. [具体风险2]
 确认跳过？"
```

## 流程位置速查

每次会话启动时，从 MEMORY.md 读取进度条：

```
PRD ✅ → 设计评审 ✅ → 原型(已冻结) ✅ → UI设计稿 ✅ → 技术方案 ✅ → 代码 ✅ → 测试验证 ✅ → 上线复盘 🔜
```

然后用可视化方式告诉用户当前状态。

---

## 常见交互模式

**用户问"接下来做什么？"**
→ 读取 MEMORY.md §2 → 对照流程表 → 回答

**用户完成一个阶段**
→ 更新 MEMORY.md §2 进度条 → 提醒流转

**用户想跳过阶段**
→ 提醒风险 → 确认 → 如确认则在 MEMORY 中记录"用户主动跳过 XX 阶段"
