# 测试一体化（Test Automation）

## 概述

手动编写测试用例耗时且容易遗漏边界场景。本 Skill 从 PRD 自动提取功能点，生成结构化测试用例矩阵和可执行的自动化测试脚本。

**核心能力**：
1. 从 PRD 提取所有功能点和交互规则
2. 生成测试用例矩阵（含正常流程、边界场景、异常场景）
3. 输出可执行的自动化测试脚本
4. 运行测试并生成验收报告

---

## 工作流程

```
PRD 文档
    │
    ▼
① 功能点提取 ──── 从 PRD 中提取所有用户故事、交互规则、边界条件
    │
    ▼
② 用例矩阵生成 ──── 每个功能点生成：正常路径 ×1 + 边界场景 ×2 + 异常场景 ×1
    │
    ▼
③ 脚本生成 ──── 生成 Python + Playwright 可执行测试脚本
    │
    ▼
④ 执行 & 报告 ──── 运行测试，输出通过率、失败详情、回归风险
```

## 测试用例矩阵格式

每个用例包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| ID | 唯一编号 | T001 |
| 分类 | 正常/边界/异常 | 边界 |
| 功能点 | 来自 PRD 的功能描述 | 打卡操作联动星星数 |
| 前置条件 | 测试前的系统状态 | 当前星星数=10，未打卡 |
| 操作步骤 | 具体操作 | 点击第一个未完成的任务 |
| 期望结果 | 预期行为 | 星星数变为 11 |
| 对应 PRD | PRD 中的需求编号 | §3.2.1 |
| 优先级 | P0/P1/P2 | P0 |

## 三条铁律

### 铁律 1：测试用例必须覆盖 PRD 所有功能点

生成用例后，对照 PRD 逐条检查，确保无遗漏。覆盖率必须 = 100%。

### 铁律 2：边界场景不能少于正常场景的 2 倍

每个功能点至少生成 2 个边界场景（如空状态、极值、并发操作）。

### 铁律 3：测试脚本必须可独立运行

每个测试用例是独立的，不依赖其他用例的执行顺序。前置条件在 setup() 中设置。
